### HTTP

HTTP — это базовый протокол, который используется для обмена информацией в Интернете. Он работает по принципу "запрос-ответ", где браузер отправляет запрос на сервер, а сервер в ответ отправляет нужные данные, например, веб-страницу, изображение или видео.

#### Основные характеристики HTTP:

- **Не обеспечивает безопасности**: Данные передаются в открытом виде, что означает, что их может перехватить и прочитать злоумышленник.
- **Статус коды**: HTTP использует статус коды для обозначения успеха или неудачи запроса. Например, 200 означает успешный запрос, 404 — страница не найдена.
- **Методы HTTP**: Включают GET (для получения данных), POST (для отправки данных), PUT (для обновления данных) и DELETE (для удаления данных).

### HTTPS

HTTPS — это защищенная версия HTTP. Он использует SSL (Secure Sockets Layer) или его более новую версию TLS (Transport Layer Security) для шифрования данных, передаваемых между клиентом и сервером.

#### Основные характеристики HTTPS:

- **Шифрование**: Все данные, передаваемые между браузером и сервером, шифруются, что делает их практически недоступными для перехвата злоумышленниками.
- **Аутентификация**: HTTPS также обеспечивает аутентификацию сервера, что помогает пользователю удостовериться, что он взаимодействует с тем сервером, с которым намеревался, а не с мошенническим сайтом.
- **Безопасность**: HTTPS защищает от атак типа "человек посередине" (man-in-the-middle), где злоумышленник может попытаться перехватить или изменить передаваемые данные.

### Разница между HTTP и HTTPS

- **Безопасность**: HTTP передает данные в открытом виде, тогда как HTTPS использует шифрование.
- **Использование**: HTTPS используется на веб-сайтах, которые требуют защиты данных, таких как интернет-магазины, онлайн-банкинг, социальные сети и другие сервисы, где обрабатываются конфиденциальные данные.
- **Скорость**: HTTPS может быть немного медленнее из-за шифрования, но современные технологии минимизируют эту разницу.

HTTP работает на 80 порте, а HTTPS на 443
