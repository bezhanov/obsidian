Ограничения (constraints) используются для указания ограничения на тип данных таблицы. Они могут быть указаны при создании или изменении таблицы. Пример ограничений:

- `NOT NULL`
- `CHECK`
- `DEFAULT`
- `UNIQUE`
- `PRIMARY KEY`
- `FOREIGN KEY`

В SQL существует несколько типов ограничений, которые помогают обеспечить целостность и корректность данных в базе данных. Вот основные типы ограничений:

### 1. **NOT NULL**
Ограничение, которое запрещает запись NULL-значений в указанный столбец.
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT NOT NULL,
      Name VARCHAR(50) NOT NULL
  );
  ```

### 2. **UNIQUE**
Гарантирует, что все значения в указанном столбце или наборе столбцов будут уникальными.
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT UNIQUE,
      Email VARCHAR(50) UNIQUE
  );
  ```

### 3. **PRIMARY KEY**
Комбинирует свойства `NOT NULL` и `UNIQUE`. Обозначает столбец или несколько столбцов, которые однозначно идентифицируют каждую запись в таблице.
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT PRIMARY KEY,
      Name VARCHAR(50)
  );
  ```

### 4. **FOREIGN KEY**
Ограничение внешнего ключа связывает одну таблицу с другой для поддержания ссылочной целостности. Значение внешнего ключа должно существовать в целевой таблице (в таблице, на которую ссылаются).
- Пример:
  ```sql
  CREATE TABLE Orders (
      OrderID INT PRIMARY KEY,
      CustomerID INT,
      FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
  );
  ```

### 5. **CHECK**
Позволяет задать условие, которому должны соответствовать данные в столбце. Если условие не выполняется, вставка или обновление записи не будет выполнено.
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT,
      Age INT CHECK (Age >= 18)
  );
  ```

### 6. **DEFAULT**
Устанавливает значение по умолчанию для столбца, если пользователь не указал значение при вставке данных.
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT,
      Name VARCHAR(50),
      Country VARCHAR(50) DEFAULT 'USA'
  );
  ```

### 7. **AUTO_INCREMENT (для MySQL)**
Используется для автоматического увеличения значения поля при добавлении новой строки (обычно используется для первичных ключей).
- Пример:
  ```sql
  CREATE TABLE Employees (
      ID INT AUTO_INCREMENT PRIMARY KEY,
      Name VARCHAR(50)
  );
  ```

### Ограничения могут применяться при создании таблицы или добавляться к существующей таблице с помощью команд `ALTER TABLE`.

- Пример добавления ограничения:
  ```sql
  ALTER TABLE Employees
  ADD CONSTRAINT UniqueEmail UNIQUE (Email);
  ```

Эти ограничения позволяют управлять целостностью данных и предотвращать некорректные изменения в базе данных.
