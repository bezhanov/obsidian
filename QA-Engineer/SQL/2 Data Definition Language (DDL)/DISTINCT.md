`DISTINCT` в SQL используется для удаления дублирующихся записей из результата запроса. Когда вы выполняете запрос с использованием `DISTINCT`, SQL-сервер возвращает только уникальные строки, исключая дубликаты. Это полезно, когда вам нужно получить уникальные значения из столбца или набора столбцов.

### Пример использования `DISTINCT`

Предположим, у вас есть таблица `employees` с такими данными:

|id|name|department|
|---|---|---|
|1|Alice|HR|
|2|Bob|IT|
|3|Alice|HR|
|4|Charlie|IT|
|5|Alice|Finance|

Запрос без `DISTINCT`:

`SELECT department FROM employees;`

Результат:

|department|
|---|
|HR|
|IT|
|HR|
|IT|
|Finance|

Запрос с `DISTINCT`:

`SELECT DISTINCT department FROM employees;`

Результат:

|department|
|---|
|HR|
|IT|
|Finance|

Здесь `DISTINCT` исключил дубликаты и оставил только уникальные значения из столбца `department`.

### Важные моменты

- `DISTINCT` применяется ко всем столбцам, указанным в SELECT.
- Если в SELECT указано несколько столбцов, то строки считаются уникальными, если все значения в указанных столбцах уникальны.
- Использование `DISTINCT` может замедлить выполнение запроса, особенно на больших таблицах, так как требуется дополнительная работа по сравнению данных на уникальность.
