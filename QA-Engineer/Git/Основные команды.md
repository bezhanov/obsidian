Git — это система контроля версий, которая позволяет отслеживать изменения в коде и управлять проектами. Вот основные команды, которые часто используются в работе с Git:

### 1. `git init`

- **Назначение**: Инициализирует новый репозиторий Git.
- **Пример**: `git init` в папке проекта создаст там пустой репозиторий.

### 2. `git clone`

- **Назначение**: Клонирует (скачивает) удаленный репозиторий на локальный компьютер.
- **Пример**: `git clone https://github.com/user/repo.git` — склонирует репозиторий.

### 3. `git add`

- **Назначение**: Добавляет файлы в индекс (стадию подготовки).
- **Пример**: `git add filename` добавляет конкретный файл. `git add .` добавляет все изменения в текущей директории.

### 4. `git commit`

- **Назначение**: Сохраняет изменения, добавленные в индекс, в историю версий.
- **Пример**: `git commit -m "Описание изменений"` — создает коммит с сообщением.

### 5. `git status`

- **Назначение**: Показывает состояние рабочего каталога и индекса.
- **Пример**: `git status` покажет, какие файлы изменены, добавлены или не отслеживаются.

### 6. `git push`

- **Назначение**: Отправляет изменения из локального репозитория в удаленный.
- **Пример**: `git push origin main` отправляет изменения в ветку `main` на удаленном репозитории.

### 7. `git pull`

- **Назначение**: Загружает изменения из удаленного репозитория и сливает их с текущей веткой.
- **Пример**: `git pull origin main` — скачивает и сливает изменения из ветки `main`.

### 8. `git branch`

- **Назначение**: Управление ветками.
- **Примеры**:
    - `git branch` — показывает список всех локальных веток.
    - `git branch branch_name` — создает новую ветку.
    - `git branch -d branch_name` — удаляет ветку.

### 9. `git checkout`

- **Назначение**: Переключение между ветками или восстановление файлов из индекса.
- **Примеры**:
    - `git checkout branch_name` — переключиться на другую ветку.
    - `git checkout -- filename` — восстановить файл из последнего коммита.

### 10. `git merge`

- **Назначение**: Сливает изменения из одной ветки в другую.
- **Пример**: `git merge branch_name` — сливает указанную ветку с текущей.

### 11. `git log`

- **Назначение**: Показывает историю коммитов.
- **Пример**: `git log` — выводит список последних коммитов с описанием.

### 12. `git remote`

- **Назначение**: Управление удаленными репозиториями.
- **Примеры**:
    - `git remote -v` — показывает список настроенных удаленных репозиториев.
    - `git remote add origin https://github.com/user/repo.git` — добавляет новый удаленный репозиторий.

### 13. `git fetch`

- **Назначение**: Загружает изменения из удаленного репозитория, но не сливает их.
- **Пример**: `git fetch origin` — загружает изменения из удаленного репозитория.

### 14. `git reset`

- **Назначение**: Отменяет изменения.
- **Примеры**:
    - `git reset HEAD filename` — убирает файл из индекса, но оставляет его изменения.
    - `git reset --hard` — откатывает все изменения в рабочей директории и индексе до последнего коммита.

### 15. `git rebase`

- **Назначение**: Переносит или изменяет базу текущей ветки на другой коммит.
- **Пример**: `git rebase branch_name` — переносит текущую ветку на базу другой ветки.

### 16. `git stash`

- **Назначение**: Временно сохраняет изменения в рабочем каталоге, чтобы можно было переключиться на другую ветку.
- **Примеры**:
    - `git stash` — сохраняет изменения и очищает рабочий каталог.
    - `git stash apply` — восстанавливает ранее сохраненные изменения.

### 17. `git diff`

- **Назначение**: Показывает различия между коммитами, файлами или ветками.
- **Пример**: `git diff` — показывает изменения, которые еще не были закоммичены.
