### Абстракция

Абстракция — это процесс выделения важных характеристик объекта и игнорирования несущественных деталей. В контексте ООП, абстракция позволяет разработчику сосредоточиться на том, что объект делает, а не на том, как он это делает. Абстракция помогает упростить сложные системы, предоставляя простой интерфейс для взаимодействия с объектами, скрывая внутреннюю реализацию.

Примером абстракции может быть интерфейс автомобиля: водителю не нужно знать, как работает двигатель, чтобы управлять машиной. Достаточно знать, как пользоваться рулем, педалями и другими элементами управления.

### Абстрактный класс

Абстрактный класс — это класс, который не может быть использован для создания объектов. Он служит основой для других классов, которые наследуются от него. Абстрактные классы часто содержат абстрактные методы — методы, которые объявляются, но не имеют реализации в абстрактном классе. Эти методы должны быть реализованы в подклассах.

Использование абстрактных классов позволяет:

1. **Создавать общий интерфейс**: Абстрактные классы задают набор методов, которые должны быть реализованы в подклассах, обеспечивая единый интерфейс для работы с различными реализациями.
    
2. **Реализовывать частичную функциональность**: Абстрактный класс может содержать как абстрактные методы (без реализации), так и обычные методы с реализацией, которые могут быть унаследованы подклассами.
    
3. **Избегать создания объектов**: Абстрактный класс не может быть инстанцирован, что предотвращает создание объектов от класса, который не предназначен для этого.
    

#### Пример на Python:

python

Копировать код

```
from abc import ABC, abstractmethod 

# Создание абстрактного класса 
class Animal(ABC):     
	@abstractmethod     
	def make_sound(self):         
		pass  

# Создание подклассов 
class Dog(Animal):     
	def make_sound(self):         
		return "Bark"  

class Cat(Animal):     
	def make_sound(self):         
		return "Meow"  
		
# Использование подклассов 
dog = Dog() 
cat = Cat() 

print(dog.make_sound())  # Вывод: Bark 
print(cat.make_sound())  # Вывод: Meow
```

В этом примере `Animal` — абстрактный класс с абстрактным методом `make_sound`, который должен быть реализован в каждом подклассе. Классы `Dog` и `Cat` наследуются от `Animal` и реализуют метод `make_sound`.

Таким образом, абстракция и абстрактные классы позволяют создавать более упорядоченные и поддерживаемые системы, предоставляя разработчикам средства для создания общих интерфейсов и разделения ответственности.
