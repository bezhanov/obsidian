- **Запуск тестов**:

    - `pytest` — Запускает все тесты в текущем каталоге и его подкаталогах.
    - `pytest <directory_or_file>` — Запускает тесты в указанном каталоге или файле.

- **Поиск тестов по имени**

    - `pytest -k "substring"` — Запускает тесты, имена которых содержат указанную подстроку. Например, `pytest -k "test_login"`.

- **Параллельный запуск тестов** (с помощью плагина `pytest-xdist`):

    - `pytest -n <num>` — Запускает тесты в несколько параллельных процессов. Например, `pytest -n 4` запустит тесты в четырех параллельных процессах.

- **Показ подробного вывода**:

    - `pytest -v` — Включает подробный вывод с именами всех тестов и их результатами.
    - `pytest -q` — Уменьшает количество выводимой информации, показывая только минимальный результат теста (успешный или нет).

- **Повторный запуск только упавших тестов**:

    - `pytest --lf` — Повторяет только те тесты, которые упали в предыдущем запуске.
    - `pytest --ff` — Повторяет сначала те тесты, которые упали в предыдущем запуске, а затем все остальные.

- **Запуск тестов с определенным маркером**:

    - `pytest -m "marker_name"` — Запускает тесты, помеченные указанным маркером. Например, `pytest -m "slow"`.

- **Создание отчета о покрытии кода** (с помощью плагина `pytest-cov`):

    - `pytest --cov=<source_dir>` — Запускает тесты и генерирует отчет о покрытии кода. Например, `pytest --cov=my_package`.

- **Прерывание при первой неудаче**:

    - `pytest -x` — Останавливает выполнение тестов после первой неудачи.
    - `pytest --maxfail=<num>` — Останавливает выполнение тестов после указанного количества неудач. Например, `pytest --maxfail=3`.

- **Пропуск тестов по условию**:

    - `pytest --skip` — Пропускает тесты, которые соответствуют определенному условию.

- **Показ вывода print() и других функций вывода**:

    - `pytest -s` — Показывает весь вывод, включая вывод от `print()`.
