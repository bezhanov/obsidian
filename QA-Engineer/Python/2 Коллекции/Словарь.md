В словаре Python ключом может быть любой неизменяемый объект, такой как число, строка, кортеж, но не изменяемый объект, такой как список.

#### Подробный ответ

Ключом в словаре Python может быть любой объект, который обеспечивает хэширование и уникальность ключа. Это означает, что объект должен быть неизменяемым, чтобы его значение хэша не менялось. Примерами таких объектов являются числа, строки и кортежи:

```
# Пример использования числа в качестве ключа
my_dict = {42: 'value'}

# Пример использования строки в качестве ключа
my_dict = {'key': 'value'}

# Пример использования кортежа в качестве ключа
my_dict = {(1, 2): 'value'}
```

Однако списки не являются хэшируемыми, поэтому они не могут быть использованы в качестве ключей в словаре:

```
# Пример невозможности использования списка в качестве ключа
my_dict = {[1, 2]: 'value'}  # Вызовет TypeError: unhashable type: 'list'
```

Пользовательские классы также могут быть использованы в качестве ключей, если они переопределяют метод `__hash__()` таким образом, чтобы он корректно вычислял хэш-значение объекта. Например:

```
class MyClass:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __hash__(self):
        return hash((self.x, self.y))

obj1 = MyClass(1, 2)
obj2 = MyClass(1, 2)

print(hash(obj1))  # Выведет хэш-значение объекта obj1
print(hash(obj2))  # Выведет хэш-значение объекта obj2
```

Здесь метод `__hash__()` определен таким образом, чтобы объекты с одинаковыми значениями x и y имели одинаковые хэш-значения.

Важно помнить, что в Python ключи в словаре должны быть уникальными, и поиск по ключу выполняется быстро благодаря хэшированию.
